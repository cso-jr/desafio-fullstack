# Desafio de código - full stack

## API Backend e Frontend

Observações:

  1. A proposta foi muito bem estruturada, com instruções claras do que deveria ser feito e como deveria ser entregue, além de explicitar quais seriam os critérios de avaliação.
  2. Entreguei incompleto. Por motivos pessoais, não pude desenvolver na totalidade, mas com calma vou concluir por aqui, aos poucos, porque é uma ótima oportunidade de aprendizado.

Dificuldades encontradas:

  1. Após a primeira leitura, o pânico se instala, a mente se fixa no que eu não conseguiria fazer de imediato e sensação de derrota é inevitável. Por sorte tenho uma mente treinada. Deixei o assunto de canto, fui fazer outras coisas, deixei o subconsciente trabalhar por alguns minutos e voltei ao computador. Tirando o que eu não sei fazer, sei fazer todo o resto, e isso é ótimo! Então, a dificuldade primeira foi entender o tamanho do problema.
  2. Dado tempo curtíssimo que eu teria, tratei de otimizar as tarefas: Backend, é que tenho mais facilidade, estruturei o projeto, criei as pastas, organizei a ordem de execução. As coisas foram ficando claras, mas então vi que mesmo no backend, a especialidade da casa, ainda tinha coisas a resolver, nebulosas, como a leitura do csv somente uma vez e então trabalhar só com os dados persistidos. Esse ponto me causou um desconforto, porque já tinha feito isso antes, mas no decorrer de um curso e sem entender o que estava fazendo. Chat GPT fez um ótimo trabalho aqui, então... Só copiei e colei, ajustei, funcionou. A partir daí não toquei mais nesse assunto. Dificuldade 2: CSV inicial para fazer o seed do banco de dados.
  3. Paginação. Não sei por que cargas d'água o Page com Pageable não funcionou. Fiz algumas tentativas com alguns arranjos e só que retornou foi quando usei List ao invés de Page. Explorar a causa-raiz toma tempo demais que eu não tinha, então resolvi entregar com List mesmo, já que podia ser alguma incompatibilidade com as bibliotecas, versões, pacotes, configurações da IDE ou do proejto, podia ser um monte de coisas que por agora não consigo esclarecer.
  4. Frontend: reconheço minha falta de habilidade com frontend. Estou cuidando dessa parte, a pokedex da DIO tem me ajudado a explorar mais recursos de HTML, CSS e consumo de api com JavaScript. Não sou habilidoso no frontend muito menos talentoso, mas sabia um pouco do que deveria fazer. O VSCode ajuda demais o desenvolvimento do front, e o plano era fazer o basicão, construir uma página com um menu lateral, usaria os recursos da fetch para trabalhar os dados do endpoint em JSon, construir o trecho do HTML dentro JavaScript, formatar com CSS, aquela coisa beeeeem básica mesmo! Nessa hora o meu foco é que a coisa funcione, mesmo feia, porque entre O Estado da Arte da Beleza que não funciona e o feio que também não funciona, o problema é o mesmo: não funciona. Ficamos assim, então... O front té no plano, assim que fizer funcionar o Page do backend, vou explorar os recursos do Javascript no front.
